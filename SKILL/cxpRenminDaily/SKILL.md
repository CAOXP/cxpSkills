# cxpRenminDaily

## 🎯 核心目标

打破时间的壁垒，让用户随时“翻阅”任意一天的人民日报。
不管是查看“今天的头条”，还是穿越回“历史上的今天”（例如十年前的今天发生了什么大事），该技能都能通过访问权威数据库（`data.people.com.cn/rmrb`），提取核心要闻，并输出结构化、易读的新闻摘要。

## 🚦 触发信号

当用户提到以下意图时，直接使用本技能：
- “看看今天的人民日报”
- “历史上的今天，人民日报头条是什么？”
- “帮我查一下 2008年8月8日 的人民日报第1版”
- “总结一下今天的人民日报要闻”
- “读报纸 / 看新闻”

## 🔍 数据源与访问逻辑

本技能强制使用人民网图文数据库进行检索，以保证历史数据的完整性和准确性。

**核心 URL 结构：**
`https://data.people.com.cn/rmrb/{YYYYMMDD}/1`
*(注：最后的 `/1` 代表第1版：要闻。)*

**执行步骤：**
1. **解析日期**：从用户输入中提取目标日期。如果用户说“历史上的今天”，则提取出当前日期的月和日，并向用户确认想要查询的年份（或默认往前推 N 年）。如果是“今天”，则使用当天的 YYYYMMDD。
2. **构建 URL**：拼接出类似 `https://data.people.com.cn/rmrb/20260403/1` 的链接。
3. **获取网页**：使用内置的 Web 访问工具访问该页面，提取出“文章标题”列表（即第1版的所有文章）。
4. **智能摘要**：将提取到的新闻标题和核心信息，结构化为易于阅读的摘要。

## 📝 默认输出结构

默认输出为精简的“读报卡片”格式：

```markdown
📰 **人民日报要闻摘要**
📅 **日期**：YYYY年MM月DD日 | 📍 **版面**：第1版 (要闻)

---

### 🌟 头条导读
- **[主标题1]**
  - *摘要：(用一两句话概括核心事件或政策导向)*
- **[主标题2]**
  - *摘要：(用一两句话概括核心事件或政策导向)*

### 📌 其它重要简讯
- [简讯标题1]
- [简讯标题2]
- [简讯标题3]

---
🔗 [查看原报纸版面](https://data.people.com.cn/rmrb/YYYYMMDD/1)
```

## 🧠 核心原则

1. **绝对忠于原文**：不编造新闻，不篡改标题，不添加主观评论。所有新闻事实必须来源于抓取到的网页内容。
2. **抓大放小**：优先提取带有“头条”、“社论（如钟声、任仲平）”特征的文章，过滤掉无意义的版面声明。
3. **遇到查无此日的处理**：如果当天无报纸（如早期某些节假日）或尚未更新，必须明确告知用户“该日期暂无数据”，并提供相近日期的建议。
4. **灵活的“历史上的今天”**：当用户查询“历史上的今天”时，可以主动挑选几个具有代表性的历史年份（如 10年前、20年前）进行对比展示，增加趣味性。
