# cxpPattenWrite ✍️🛡️

> **辅助研发人员，将技术成果转化为严谨的专利交底书。**
> 
> 这是一个旨在帮助工程师和研究员梳理专利逻辑的辅助 Skill。它不负责扩充篇幅，而是致力于协助您将日常的技术积累，转化为符合专利规范的保护点。

## 💡 技术与专利的差异

在撰写专利时，我们经常会遇到技术视角与专利视角不一致的情况：

1. **“代码跑通/实验成功，就可以直接申请专利。”** 
   *说明：* 专利体系通常不直接保护纯代码或自然规律。专利保护的更多是**“为了解决某个具体问题，而提出的一套可实施的技术方案”**。
2. **“将已发表的学术论文直接修改格式进行申请。”** 
   *说明：* 论文的目的在于**“公开研究发现并证明其有效性”**，而专利的目的在于**“界定清晰的保护范围”**。论文注重详尽的机理与证明，专利则需要在“技术公开充分”和“保护范围合理”之间寻找平衡。
3. **“直接将会议纪要或思维导图交给代理师。”** 
   *说明：* 代理师精通专利法，但可能需要更多时间理解您的前沿技术。如果您能提前梳理出核心的技术主线，将有助于代理师更准确地把握保护重点。

**`cxpPattenWrite` 的初衷，就是协助您跨越这些视角差异。**

## 🛑 概念参考：发明、实用新型与学术论文

在开始梳理材料之前，建议先明确当前想法所属的类别：

| 维度 | 💡 发明专利 | 🛠️ 实用新型 | 📄 学术论文 |
| --- | --- | --- | --- |
| **核心目的** | 保护**“如何实现”** (流程、方法、算法协同等) | 保护**“产品结构”** (物理形状、构造组合等) | **解释并证明**研究发现或学术观点 |
| **关注重点**| 步骤、控制逻辑、材料工艺、系统数据流 | 物理形态优化、装置改进、部件连接关系 | 机理分析、实验设计、结果讨论 |
| **写作主线** | 问题 → 方案 → 实施例 → 保护点布局 | 结构组成 → 连接关系 → 部件作用 → 效果 | 背景 → 方法 → 实验 → 结果 → 讨论 |
| **细节处理**| 需保证可实施性，同时考虑合理的变形空间 | 结构需具体明确，强调物理上的可制造性 | 追求详尽严谨，注重逻辑与数据证明 |

**简而言之：** 
- 论文侧重于 **“讲明白并证明”**。
- 专利侧重于 **“讲清楚并合理保护”**。
- 发明专利偏重 **逻辑与流程**，实用新型偏重 **实体与结构**。

## ✨ 这个 Skill 能提供哪些协助？

`cxpPattenWrite` 借鉴了专利代理行业的常见规范，帮助您将零散的材料整理入专业的框架中：

- 🌪️ **材料梳理**：协助将聊天记录、零散笔记、会议纪要转化为具有逻辑主线的交底书草案。
- 🔍 **要点提取**：尝试从您的 Markdown/Docx 草稿中，提炼出潜在的技术保护点。
- 🏗️ **逻辑完善**：检查方案的完整性（如是否缺少实施例、系统方案或背景技术对比），并提醒补充，**坚持基于事实，不虚构技术细节**。
- 🎯 **信息整合**：根据方法、技术、系统三个层级，对多份参考材料进行结构化重组。

## 🧠 核心梳理逻辑：分层与递进

在整理交底书时，我们建议采用以下逻辑框架：

### 1. 清晰的技术主线
**现有问题 → 核心方案 → 技术细节展开 → 系统整合 → 预期效果**
(建议所有内容都围绕这条主线展开，保持紧凑)

### 2. 保护点的“三层结构”
- **方法层**：关注步骤、顺序、条件、输入输出、控制逻辑。
- **技术层**：关注材料、结构、工艺、算法、参数、模块交互关系。
- **系统层**：关注整体系统组成、模块接口、数据流转、协同机制。

### 3. 实施例的“递进展示”
1. **基础实施例**：能实现目的的最小可行方案（MVP）。
2. **增强实施例**：补充材料、结构、工艺、控制或算法等具体细节。
3. **集成实施例**：展示完整的系统、业务流程或工程化落地的应用场景。

### 场景二：多份材料整合 (综合输入)
```markdown
材料 1：思考记录（提取问题和方案）
材料 2：整理要求（请按固定章节归类）
材料 3：补充说明（关于实施例和保护点的扩展）
请以材料 1 为主线，参考材料 2、3，整理成一份清晰的专利交底书草案。
```

### 场景三：严谨整理模式 (仅归类，不扩写)
```markdown
以下内容仅需进行格式整理和逻辑归类，请勿添加原文未提及的技术事实。
请按标准结构分类；信息缺失的章节请标注 [待补充]；难以归类的内容统一放至“未归类信息”。
```

### 场景四：方案扩展与完善 (深度分析)
```markdown
基于当前内容，请协助完成以下分析：
1. 提炼核心方法步骤
2. 梳理相关的扩展技术方案
3. 尝试组织系统级实施例与保护点，并提示目前还缺失哪些必要数据或说明。
```
